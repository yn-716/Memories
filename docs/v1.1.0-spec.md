最終更新: 2026-07-01
基準タグ: `1.1.0`

## 2. v1.1.0 の対象範囲

v1.1.0 では以下を提供します。

- ホーム画面の「うちの子カレンダー」導線
- 月表示のカレンダー画面
- 日付ごとの写真登録、写真位置調整、天気アイコン登録
- カレンダー画像のプレビュー、保存、共有
- ホーム画面Widgetの `systemSmall` / `systemMedium` / `systemLarge`
- カレンダー状態変更時のWidget用事前レンダリング画像更新

v1.1.0 では以下を提供しません。

- ロック画面Widget
- まとめて登録
- 天気API連携
- 写真ライブラリ全体スキャン
- Widgetからの写真ライブラリ読み取り

## 3. ホーム導線

HomeView には、既存の「画像編集」と同じ主ボタン系の見た目で「うちの子カレンダー」ボタンを表示します。

- 「画像編集」ボタン: 写真選択前にスタイル選択を開く
- 「うちの子カレンダー」ボタン: `PetCalendarHomeView` を開く
- カレンダー導線は緑系グラデーションで、画像編集ボタンと区別する
- Deep Link `memories://calendar` はカレンダー画面を開く
- Deep Link `memories://calendar/today` はカレンダー画面を開いたあと今日の編集画面を開く

Deep Link は allow-list 方式です。登録、削除、購入、エクスポート、ファイル操作をURLから実行してはいけません。

## 4. うちの子カレンダー画面

カレンダー画面は月単位で表示します。

- 月移動: 前月 / 翌月ボタン
- 表示言語: 日本語 / English の segmented picker
- 週開始: 日曜固定
- 日付セル: 当月の日付のみタップ可能
- 未来日: 表示はするが登録不可
- 前月・翌月の日付: グリッド調整用に必要な範囲だけ表示し、余分な翌月週は表示しない
- 説明文: 「日付をタップして写真や天気アイコンを登録できます。」
- 「今月N枚」「連続N日」「今日の思い出を編集」は表示しない
- 「まとめて登録」は導線・実装ともに存在しない

月グリッドは `PetCalendarDateRules.monthGrid` を使います。表示行数は月初・月末の曜日から必要な 4-6 週分になり、常に6週固定にはしません。

## 5. 日別編集

日付をタップすると `PetCalendarDayEditorView` を開きます。

### 5.1 登録条件

- 登録可能なのは今日以前の日付のみ
- 写真または天気アイコンのどちらかが必要
- 写真なし、天気アイコンのみの登録を許可する
- 既存写真がある日に新しい写真を選んだ場合は置き換え確認を表示する
- 写真の撮影日が選択日と異なる場合、選択日または撮影日への登録を選ばせる
- 撮影日が未来日の場合、撮影日への登録選択肢は出さない

### 5.2 写真編集

- 写真プレビュー上でドラッグして位置を調整できる
- ピンチで拡大率を調整できる
- リセットで `PhotoPlacement.default` に戻す
- `PhotoPlacement` は保存時に `clamped` を適用する
- App内カレンダー、日別編集プレビュー、カレンダー出力、Widgetは同じ `PhotoPlacementLayout.drawRect` を使って写真位置を揃える
- 写真選択ボタンと登録ボタンは同じ行に並べる
- 写真選択ボタンは淡いカード風、登録ボタンは主ボタン風にして誤操作を避ける

### 5.3 天気アイコン

カレンダーの日付セルには天気アイコンだけを載せられます。

- アイコン: 晴れ / くもり / 雨 / 雪
- SF Symbols: `sun.max` / `cloud` / `cloud.rain` / `snowflake`
- 表示位置: 左上 / 右上 / 左下 / 右下
- 色: 白 / 黒 / ネイビー / ブルー / ソフトグレー
- 初回有効化時に色が白のままなら、視認性確保のためブルーへ変更する
- すでにユーザーが色を編集している場合、天気アイコンを変更しても色は保持する
- 天気アイコンを非表示にしても、選択済みアイコン・色・位置はモデル上保持する

## 6. 保存モデル

主なモデルは以下です。

- `PetCalendarDayEntry`
- `PhotoPlacement`
- `PetCalendarOverlayStyle`
- `PetCalendarWidgetSnapshot`
- `PetCalendarWidgetRenderedImageSet`

`PetCalendarDayEntry` は日付ID、日付、画像ファイル名、サムネイルファイル名、写真位置、天気アイコン設定、作成日時、更新日時を持ちます。caption は互換性のため残っていますが、現在の実装では空文字として保存され、表示・レンダリングには使いません。

保存先:

- App本体データ: Application Support / `Memories/PetCalendar`
- 原寸表示用画像: `PetCalendar/Images`
- サムネイル: `PetCalendar/Thumbnails`
- index: `PetCalendar/index.json`
- Widget用データ: App Group / `PetCalendarWidget`
- Widget snapshot: `PetCalendarWidget/widget-snapshot.json`
- Widget用JPG: `PetCalendarWidget/pet-calendar-widget-*.jpg`

画像保存:

- 表示用画像は長辺最大 `1600px`、JPEG quality `0.82`
- サムネイルは長辺最大 `400px`、JPEG quality `0.76`
- Widget画像は各サイズで事前レンダリングし、JPEG quality `0.84`

ファイル保護:

- App本体のカレンダーデータは `.complete`
- Widget用snapshot/JPGは `.completeUntilFirstUserAuthentication`

過去にApp Group直下の `PetCalendar` に保存されていたデータがある場合、初期化時にApplication Supportへ移行します。移行に成功した場合は旧 `index.json`、旧 `Images`、旧 `Thumbnails` を削除します。移行に失敗した場合は旧データを残します。

## 7. カレンダー出力

`PetCalendarPreviewView` は選択月のカレンダー画像を生成し、保存または共有できます。

- 既定出力サイズ: `1600 x 2200`
- 背景: 白基調の透明ガラス風パネル
- 日付セル: App内カレンダーと同じ写真位置ロジック
- 天気アイコン: 日別編集で指定したアイコン、色、位置を反映
- ウォーターマークあり/なしを選択可能
- 保存は写真ライブラリへの追加保存のみ
- 共有はShare Sheetを使う

ウォーターマーク仕様:

- 表示文字列: `Memories Pet Life`
- 可能な場合は `watermark_app_icon` を併記する
- 無料ユーザーはウォーターマークありを無制限で使える
- 無料ユーザーのウォーターマークなし出力は1日1回
- 7日間パスまたは永久パスが有効なユーザーはウォーターマークなしを無制限で使える
- ウォーターマークなし共有はShare Sheet完了後に無料枠を消費する

## 8. Widget仕様

Widgetはホーム画面Widgetのみ対応します。ロック画面Widgetファミリーは `supportedFamilies` に含めません。

Widget設定:

- Widget kind: `MemoriesWidget`
- 表示名: `Memories Pet Life`
- 説明: `Pet Calendar`
- 対応サイズ: `systemSmall`, `systemMedium`, `systemLarge`
- `systemSmall` タップ時: `memories://calendar/today`
- `systemMedium` / `systemLarge` タップ時: `memories://calendar`

WidgetはSwiftUIでカレンダーを組み立てず、App本体が生成したJPGを表示します。これによりWidget側の処理を軽量化し、写真・文字・天気アイコンの表示崩れを避けます。

### 8.1 Widget画像

生成サイズ:

- Small: `540 x 540`
- Medium: `1080 x 510`
- Large: `1080 x 1080`

表示内容:

- Small: 今日のカード。今日の写真がある場合は写真を全面表示し、日付・曜日・天気アイコン・ウォーターマークを重ねる。今日の写真がない場合、前日の写真へフォールバックしない。
- Medium: 今日を含む1週間の横長カレンダー。月全体を縦圧縮しない。
- Large: 選択月の月間カレンダー。必要な週数だけ表示する。

Widget内ウォーターマーク:

- `showsBranding == true` のとき表示する
- 表示文字列は `Memories Pet Life`
- 可能な場合は `watermark_app_icon` を使う
- Widget表示だけでは無料のウォーターマークなし枠を消費しない
- パスが有効で、最新の権利状態で `hasUnlimitedAccess == true` の場合は非表示にする

### 8.2 Snapshot

Widget snapshot JSONには以下のみを保存します。

- `updatedAt`
- `selectedMonth`
- `displayLanguage`
- `showsBranding`
- `renderedImageSets`

Widget snapshot JSONには、App本体の写真ファイル名、サムネイルファイル名、caption、`PhotoPlacement`、`PetCalendarOverlayStyle` などの編集詳細を含めません。Widgetはsnapshotに記載された事前レンダリング済みJPGだけを読みます。

`renderedImageSets` は今日と明日の2日分を持ちます。Widgetは表示時点の日付IDに一致する画像セットを選びます。これにより日付が変わった直後でも、今日が未登録なら前日の写真を表示しません。

### 8.3 更新タイミング

App本体は以下のタイミングでWidget snapshotとJPGを更新します。

- App起動時
- カレンダー画面の表示・再読み込み時
- カレンダー表示言語の変更時
- 課金状態の更新時
- 日別エントリの追加・更新・削除時

更新後は `WidgetCenter.shared.reloadTimelines(ofKind: "MemoriesWidget")` を呼び、さらに約0.4秒後に再度reloadします。

Widget側のTimelineは以下の早い方で次回更新されます。

- 15分後
- 次の日の 00:01

### 8.4 古いWidget画像の削除

Widget画像を再生成したあと、現在のsnapshotが参照していない `pet-calendar-widget-*.jpg` は削除します。生成中にエラーが起きた場合、その処理で作成したJPGを削除してからエラーにします。

## 9. 権利状態とウォーターマーク

課金商品:

- 7日間パス: `memories.7day.pass`
- 永久パス: `memories.lifetime.pass`

権利状態:

- StoreKitの現在のEntitlementから更新する
- 7日間パスは購入日時から7日間有効
- 永久パスは有効なtransactionがある限り有効
- 権利確認の鮮度上限は6時間
- 鮮度が古い場合、無制限のウォーターマークなし権利としては扱わない

カレンダー出力とWidgetの違い:

- カレンダー出力のウォーターマークなしは無料枠またはパス状態を見て制御する
- Widgetのウォーターマーク有無は `showsBranding` に事前反映する
- Widget表示では無料枠を消費しない

## 10. セキュリティ・プライバシー

- Widgetが読むのはApp Group内の `PetCalendarWidget/widget-snapshot.json` と事前レンダリングJPGのみ
- App本体の `PetCalendar/index.json`、原画像、サムネイルはApplication Supportに保存する
- Widgetは写真ライブラリへアクセスしない
- カレンダー出力の保存は `PHPhotoLibrary.requestAuthorization(for: .addOnly)` を使う
- Deep Linkは `memories://calendar` と `memories://calendar/today` のみ許可する
- Deep Linkから登録、削除、購入、エクスポート、ファイル操作を実行しない
- カレンダー登録では未来日を拒否する
- サーバーレス構成のため、カレンダーデータは端末内とApp Group内に閉じる

## 11. 手動確認項目

リリース前に以下を確認します。

- App target と Widget target のApp Groupsに `group.com.myfs716.Memories` が入っている
- App target に URL Scheme `memories` がある
- `systemSmall` / `systemMedium` / `systemLarge` のホーム画面Widgetを追加できる
- ロック画面Widgetが追加候補に出ない
- 今日の写真登録後、Small Widgetが今日の写真を表示する
- 今日未登録かつ前日登録済みの状態で、Small Widgetが前日の写真を表示しない
- Medium Widgetが1週間表示になっている
- Large Widgetが月間表示になっている
- 天気アイコンのみの登録がApp内、出力、Widgetに反映される
- 天気アイコン変更時に編集済みの色が保持される
- カレンダー表示言語変更後、Widgetにも言語が反映される
- 無料ユーザーではWidgetにウォーターマークが表示される
- パス有効ユーザーではWidgetのウォーターマークが非表示になる
- Widget画像更新後、古い `pet-calendar-widget-*.jpg` が残り続けない
- カレンダー保存・共有時のウォーターマークあり/なし制御が課金状態と一致する
