# v1.2.0 Announcement JSON Client

Last updated: 2026-07-01

## Scope

The iOS app reads public announcement data from:

```text
https://memories.myfs716.com/announcements.json
```

Cloudflare Pages deployment and production JSON updates are handled outside this app repository. The app repository only implements fetching, caching, filtering, display, and read-state management.

## Data Flow

1. The app requests the public JSON endpoint.
2. The response is decoded into announcement records.
3. The latest successful response is cached locally.
4. Announcements are filtered on device by `isActive`, `expiresAt`, `minimumAppVersion`, `maximumAppVersion`, and the local announcement display baseline.
5. The current display language selects Japanese or English text, with fallback to the other language when needed.
6. Read IDs are stored locally and used to calculate the unread badge count.
7. The unread badge is calculated only from announcements that remain visible after all filters.

## Refresh Policy

- App launch: refresh only when the previous automatic refresh is older than 6 hours.
- Foreground resume: refresh only when the previous automatic refresh is older than 6 hours.
- Home display: refresh only when the previous automatic refresh is older than 6 hours.
- Manual refresh: ignore the 6-hour throttle.

If fetching fails and cached announcements exist, the app continues to show the cached data.

## Privacy and Network Behavior

- No login is required.
- The app does not upload photos.
- The app does not upload videos.
- The app does not upload edits.
- The app does not upload drafts.
- The app does not upload generated images or videos.
- The app does not upload user-generated content.
- The request URL does not include user ID, device ID, advertising ID, or app-generated user identifiers.
- Language and version targeting are not sent as request parameters; filtering happens on device.

## Supported JSON Shape

```json
{
  "announcements": [
    {
      "id": "2026-07-01-v120",
      "titleJa": "動画編集に対応しました",
      "bodyJa": "写真だけでなく、動画にもカード風フレームを重ねられるようになりました。",
      "titleEn": "Video editing is now supported",
      "bodyEn": "You can now create card-style videos as well as photos.",
      "publishedAt": "2026-07-01T00:00:00Z",
      "expiresAt": "2026-09-01T00:00:00Z",
      "isActive": true,
      "url": "https://memories.myfs716.com/support/",
      "minimumAppVersion": "1.2.0",
      "maximumAppVersion": null,
      "priority": 0,
      "showForNewUsers": false
    }
  ]
}
```

`expiresAt` is optional and may be `null`. `showForNewUsers` is optional and defaults to `false` when omitted.

## Display Baseline

The app stores a local announcement display baseline timestamp the first time the announcement feature is initialized. The value is stored in UserDefaults and is not overwritten on later launches.

For users who update to v1.2.0 and do not already have this value, the app initializes it at that update-time launch. The app does not attempt to infer the exact historical install date.

Announcements published before this baseline are hidden unless `showForNewUsers` is `true`.

## Expiration Filtering

If `expiresAt` exists and is earlier than the current device time, the announcement is hidden. Missing or `null` `expiresAt` means the announcement does not expire.

## Version Filtering

`minimumAppVersion` and `maximumAppVersion` are compared using semantic-style numeric components, not plain string comparison. For example, `1.2.10` is treated as newer than `1.2.2`.

## New User Filtering

Old update announcements should generally use `showForNewUsers: false` with an `expiresAt` date. This prevents old release notes from appearing as unread news for people who install the app later.

Example update announcement:

```json
{
  "id": "2026-07-01-v120",
  "titleJa": "動画編集に対応しました",
  "bodyJa": "写真・動画編集に対応しました。",
  "titleEn": "Video editing is now supported",
  "bodyEn": "Photo and video editing is now supported.",
  "publishedAt": "2026-07-01T00:00:00Z",
  "expiresAt": "2026-09-01T00:00:00Z",
  "isActive": true,
  "url": null,
  "minimumAppVersion": "1.2.0",
  "maximumAppVersion": null,
  "priority": 0,
  "showForNewUsers": false
}
```

Important announcements can opt in to new-user visibility with `showForNewUsers: true`. They may omit `expiresAt` or set it to `null` when the information should remain visible.

Example important announcement:

```json
{
  "id": "2026-07-15-important",
  "titleJa": "保存機能についてのお知らせ",
  "bodyJa": "一部環境で保存に時間がかかる場合があります。",
  "titleEn": "Notice about saving",
  "bodyEn": "Saving may take longer on some devices.",
  "publishedAt": "2026-07-15T00:00:00Z",
  "expiresAt": null,
  "isActive": true,
  "url": null,
  "minimumAppVersion": null,
  "maximumAppVersion": null,
  "priority": 10,
  "showForNewUsers": true
}
```

## UI Behavior

- Home shows a bell button.
- The bell badge count is the number of visible announcement IDs not present in the local read-ID set.
- Opening an announcement marks it as read.
- The announcements screen includes a "Mark All as Read" action.
- Announcements with a URL show a "Learn More" action.
- URL opening uses the app's safe Safari presentation.

## Repository Boundary

This app repository must not perform Cloudflare Pages deployment, create production Pages files, or update support/privacy/legal pages as part of v1.2.0 app implementation.
