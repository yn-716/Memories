# Memories Pet Life v1.2.0 Release Notes

Last updated: 2026-07-01

## Summary

v1.2.0 expands the editor from photo-only editing to photo/video editing, adds in-app announcements from a public JSON endpoint, improves calendar deletion safety, fixes the Settings version display, and introduces a StoreKit review request flow after successful saves.

## Version

- Marketing Version: `1.2.0`
- Current Project Version: `1`
- Settings screen display: `1.2.0` only

The build number is intentionally hidden from users.

## Changes

- Photo/video editor
  - Home now opens a photo/video picker.
  - Images keep the existing editing, save, share, draft, and watermark behavior.
  - Videos can use all existing editor templates: ticket frame, simple card, and retro film.
  - Video preview loops muted in the editor and preview screens.
  - Video export uses AVFoundation and preserves source audio when available.
  - Video placement uses the same scale and offset editing as photos.

- Storage and cleanup
  - Temporary imported and exported videos are stored under temporary/cache locations.
  - Temporary media files are excluded from iCloud backup.
  - Draft videos are stored locally as app working data and excluded from iCloud backup.
  - Deleting a video draft removes its video and thumbnail files.
  - App launch and editor/preview flows clean old temporary files.
  - Settings includes a "Delete Temporary Files" action that does not delete drafts.

- Calendar delete confirmation
  - Existing calendar entries now show a destructive delete button.
  - Delete requires confirmation before repository deletion, widget snapshot update, timeline reload, `onSaved()`, and dismiss.
  - Delete is disabled while processing.

- Announcements
  - Home shows an announcement bell.
  - Announcements are loaded from `https://memories.myfs716.com/announcements.json`.
  - The app caches the last successful JSON response and stores read IDs locally.
  - Automatic refresh is limited to once every 6 hours.
  - Manual refresh is available from the announcements screen.
  - Active, version-targeted, non-expired, and new-user-appropriate announcements are filtered on device.
  - Old update announcements can be hidden from new users with `showForNewUsers: false`, while important notices can opt in with `showForNewUsers: true`.

- Review request
  - StoreKit review requests are considered only after successful photo save, video save, or calendar image save.
  - Draft saves, share completions, failed saves, app launch, and editor open do not trigger review requests.
  - The default threshold is the third successful save.
  - Requests are rate-limited to once per app version and at least 50 days apart.

- Localization
  - Japanese and English strings are provided for new editor, announcement, calendar delete, settings cleanup, and save/share states.

## Privacy Notes

- The app does not upload user photos, videos, edits, drafts, or generated media to the announcement endpoint.
- The app does not attach user ID, device ID, advertising ID, or user-specific query parameters to the announcement JSON request.
- Announcement language and version filtering are performed on device after downloading the public JSON.
- The app repository does not deploy or modify Cloudflare Pages content for v1.2.0.
